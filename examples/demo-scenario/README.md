# Demo Scenario: ResumeScreen AIMS

A realistic end-to-end scenario exercising every AIGovOps plugin against a single AI system. Intended as the reference "hello world" for anyone evaluating the catalogue.

**Scenario:** a mid-sized professional-services firm deploys `ResumeScreen`, an internal AI tool that ranks candidate resumes against a job posting for human reviewer consideration. The organization is operating under ISO/IEC 42001:2023 and adopting NIST AI RMF 1.0 alignment for US customer assurance.

## What this demo produces

Running `python run_demo.py` executes all nine plugins in sequence against the scenario inputs in `inputs/` and writes all artifacts to `outputs/`. At the end, a composite `summary.md` links to every artifact and describes what the AIMS state looks like for this system.

## Files

| Path | Purpose |
|---|---|
| `inputs/ai_system_inventory.json` | One-system inventory. |
| `inputs/risks.json` | Risks identified through stakeholder consultation. |
| `inputs/org_chart.json` | Organizational roles. |
| `inputs/role_assignments.json` | RACI matrix. |
| `inputs/authority_register.json` | Authority basis per role. |
| `inputs/measurements.json` | Precomputed trustworthy-AI measurements. |
| `inputs/thresholds.json` | Organizational threshold policy. |
| `inputs/stakeholders.json` | Affected stakeholder groups for the AISIA. |
| `inputs/impact_assessments.json` | Pre-populated AISIA sections. |
| `inputs/governance_decisions.json` | Governance events to log. |
| `run_demo.py` | Orchestrates the flow. |
| `outputs/` | Every artifact generated by the run (gitignored after generation). |

## Running the demo

```bash
cd examples/demo-scenario
python3 run_demo.py
```

Output lands in `outputs/`. The script is idempotent: running it twice overwrites the prior run's outputs.

## What the demo exercises

- `audit-log-generator` produces an entry for the deployment governance event.
- `role-matrix-generator` produces the RACI matrix from the supplied org chart and role assignments.
- `risk-register-builder` enriches the supplied risks with scoring and control linkage.
- `soa-generator` produces the Statement of Applicability using the risk register's existing-control references.
- `aisia-runner` produces the AI System Impact Assessment.
- `nonconformity-tracker` processes a demonstration nonconformity through the Clause 10.2 lifecycle.
- `management-review-packager` assembles the Clause 9.3.2 input package from all of the above.
- `metrics-collector` processes the supplied measurements and flags any threshold breaches.
- `gap-assessment` assesses the current state against iso42001.

## Use this demo for

- Evaluating plugin output quality before running against real organizational data.
- Training a new governance team member on the end-to-end artifact flow.
- Regression testing when plugin versions change (compare outputs across runs).
- Producing screenshots or walkthroughs for documentation.

## What this demo does NOT do

- It does not run Hermes Agent. The demo invokes plugins directly in Python. To run the same flow through the AIGovClaw runtime, install AIGovClaw (see the aigovclaw repository) and invoke the workflows instead.
- It does not push artifacts to any external GRC platform. Outputs land in the local `outputs/` directory. Adapter-based push is Phase 4.
- It does not replace organizational input. The scenario is a plausible composite; replace the inputs with your own organization's data for real-world use.
